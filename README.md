# IS Project Management Practice

## 📌 Project Overview
This project demonstrates the use of project management tools, specifically Trello and GitHub, to manage tasks, track progress, and support team collaboration in an Information Systems (IS) project.

---

## 🎯 Objectives
- Apply Trello for task management
- Use GitHub for version control and collaboration
- Organize project activities effectively
- Improve teamwork and communication

---

## 🛠 Tools Used
- Trello (Task Management)
- GitHub (Version Control & Collaboration)

---

## 📂 Project Files
The repository contains the following documents:
- Project Proposal
- Project Plan
- Risk Management Document
- Final Report

---

## 📋 Project Tasks
- Define project scope  
- Identify stakeholders  
- Create Work Breakdown Structure (WBS)  
- Prepare risk analysis  
- Prepare project schedule  
- Write final report  

---

## 🔄 Workflow
Tasks were managed in Trello using the following stages:
- To Do  
- In Progress  
- Testing  
- Completed  

---

## 🤝 Collaboration
- Team members contributed by uploading and updating files
- GitHub commits were used to track changes
- Issues were created to manage and assign tasks

---

## ✅ Benefits
- Improved task organization  
- Better team collaboration  
- Clear progress tracking  
- Effective version control  

---

## 📌 Conclusion
This project shows how Trello and GitHub can be effectively used together to manage IS projects, improve productivity, and ensure successful teamwork.
